## **\_\_name\_\_**

__name__은 파이썬이 내부적으로 관리하는 내장 특수 변수(Built-in Variable)이다.

현재 실행 중인 파일이 '직접 실행'되었는지, 아니면 다른 파일에 의해 'import(모듈로 호출)'되었는지 구분할 때 사용한다.

---

### **(1) 기본 구조**

```python
if __name__ == "__main__":
    # 이 파일이 터미널에서 직접 실행되었을 때만 작동할 코드

```

* **직접 실행 시**: 파이썬은 해당 파일의 `__name__` 변수에 `"__main__"`이라는 값을 자동으로 부여한다.
* **import 되어 실행 시**: 해당 파일의 `__name__` 변수에는 `"원래의 모듈 이름"`이 들어간다.

---

### **(2) 왜 사용할까? (문제 상황 가정)**

다른 파일에서 계산용 함수만 빌려 쓰려고 `import`했는데, 모듈 파일 안에 적어둔 테스트용 실행 코드까지 멋대로 같이 실행되는 것을 막기 위해 사용한다.

우선 아래와 같은 모듈 파일(`calc.py`)이 있다고 가정한다.

```python
# calc.py
def add(a, b):
    return a + b

# 모듈 개발 중에 잘 작동하는지 확인해본 테스트 코드
print("calc.py 결과:", add(10, 20))

```

이 상태에서 `main.py`가 `calc` 모듈을 불러온다.

```python
# main.py
import calc

print("main.py가 실행되었습니다.")

```

`main.py` 실행 결과:

```python
calc.py 결과: 30
main.py가 실행되었습니다.

```

`main.py`는 단지 `add` 함수 기능만 필요해서 import 했을 뿐인데, `calc.py` 내부에 있던 `print()` 코드까지 통째로 실행되어 버린다.

---

### **(3) 해결 방법 (if __name__ == "__main__":)**

모듈 파일(`calc.py`)의 테스트 코드를 특수 조건문으로 감싸서 보호한다.

```python
# calc.py (수정본)
def add(a, b):
    return a + b

if __name__ == "__main__":
    # calc.py를 직접 실행(Run)할 때만 이 안의 코드가 동작함
    print("calc.py 직접 실행 테스트:", add(10, 20))

```

이제 다시 `main.py`를 실행하면 원치 않는 테스트 코드는 실행되지 않는다.

`main.py` 실행 결과:

```python
main.py가 실행되었습니다.

```

---

### **(4) 변수 값의 변화 원리**

파이썬 파일이 구동되는 실행 주체에 따라 `__name__`에 할당되는 문자열이 달라진다.

```text
[ calc.py를 마우스 우클릭 후 직접 Run 했을 때 ]
calc.py 안의 __name__ 변수 ──> "__main__" 할당 (조건문 통과, 테스트 실행 O)

[ main.py를 실행하고 calc.py는 import만 되었을 때 ]
calc.py 안의 __name__ 변수 ──> "calc" (모듈명 할당, 조건문 탈락, 테스트 실행 X)
main.py 안의 __name__ 변수 ──> "__main__" 할당

```

---

### 정리

- __name__은 파이썬 파일의 실행 경로 및 정체성을 담고 있는 내장 변수이다.
- 스크립트 파일을 터미널에서 직접 실행하면 `__name__` 값은 무조건 `"__main__"`이 된다.
- 외부에서 모듈로 `import`될 경우 `__name__` 값은 원래의 `"파일 이름"`이 된다.
- `if __name__ == "__main__":` 문법을 사용하면 모듈 파일 내의 독립적인 테스트 코드가 다른 파일에 의해 오염·유출되는 것을 완벽히 방지할 수 있다.
