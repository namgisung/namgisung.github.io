## **다중 상속과 접근 제한자**

파이썬은 하나의 자식 클래스가 둘 이상의 부모 클래스로부터 동시에 속성과 메서드를 물려받는 **다중 상속(Multiple Inheritance)**을 지원한다.

---

### **(1) 단일 상속 vs 다중 상속의 생성자 호출 방식 차이**

자식 객체가 생성될 때 부모의 생성자(`__init__`)를 호출하는 방식은 상속 구조에 따라 명확하게 구분하여 코딩해야 한다.

#### **1. 단일 상속인 경우**
`super().__init__()` 문법을 사용하여 직계 부모의 생성자를 간결하게 호출한다.
```python
class Child(Parent):
    def __init__(self):
        super().__init__()  # 단일 상속은 super() 하나로 해결

```

#### **2. 다중 상속인 경우**

여러 부모가 존재하므로 명확한 구분을 위해 **`부모클래스명.__init__(self, ...)`** 형식을 사용하여 명시적으로 어떤 부모의 생성자를 호출할지 지정한다. 이때 첫 번째 인자로 `self`를 반드시 수동으로 전달해야 한다.

```python
class Child(Parent1, Parent2):
    def __init__(self):
        Parent1.__init__(self)  # Parent1 생성자 호출 (self 명시 필수)
        Parent2.__init__(self)  # Parent2 생성자 호출 (self 명시 필수)

```

---

### **(2) 다이아몬드 상속과 MRO 선착순 법칙**

다중 상속 구조에서 발생할 수 있는 가장 대표적인 혼란 상황을 다이아몬드 상속(Diamond Inheritance)이라고 한다.

#### **문제 상황**

만약 자식 클래스 `D`가 부모 클래스 `B`와 `C`를 동시에 상속받았는데, 두 부모가 모두 상위 클래스 `A`로부터 물려받은 동일한 이름의 `method()`를 오버라이딩하고 있다면, `D`는 과연 누구의 메서드를 물려받게 되는 걸까?

```python
class A:
    def method(self):
        print("A")

class B(A):
    def method(self):
        print("B")

class C(A):
    def method(self):
        print("C")

class D(B, C):
    pass

obj = D()
obj.method()

```

결과:

```text
B

```

#### **파이썬의 해결책: MRO 선착순 구조**

파이썬은 이러한 다이아몬드 상속의 모호성을 해결하기 위해 **MRO(Method Resolution Order, 메서드 탐색 순서)** 규칙을 설계했다.

- 자식 클래스를 선언할 때 괄호 안에 적어준 부모 클래스 목록에서 **맨 앞(좌측)에 명시된 클래스가 최고의 우선권**을 가진다.
- `class D(B, C):` 형태로 정의했기 때문에, 좌측에 적힌 `B`가 `C`보다 우선순위가 높아 `B` 내부의 메서드가 실행된다.
- 만약 코드를 `class D(C, B):` 로 변경한다면 우선순위가 뒤바뀌어 결과는 `C`가 출력된다.

### **(3) 인스턴스 변수의 접근 제한자 (Access Specifier)**

파이썬은 다른 프로그래밍 언어(Java, C++ 등)와 달리 `public`, `private` 같은 키워드가 존재하지 않는다. 대신 오직 **변수 이름 앞에 붙는 언더바(`_`)의 개수**로 데이터의 접근 범위를 조절한다.

```python
class AccessExample:
    def __init__(self):
        self.public_val = "누구나 접근 가능 (Public)"
        self._protected_val = "상속 관계/자식끼리만 쓰자고 약속 (Protected)"
        self.__private_val = "외부 및 자식 접근 절대 금지 (Private)"

```

- **Public Field (`self.변수`)**
- 아무런 제한이 없다. 클래스 외부, 내부, 자식 클래스 어디서든 자유롭게 읽고 수정할 수 있다.


- **Protected Field (`self._변수`)**
- 이름 앞에 언더바 **1개**를 붙인다. 문법적으로 강제 제한되지는 않으나, 개발자 간의 약속으로 "외부에서 직접 건드리지 말고 상속받은 자식 클래스 내부에서만 활용하라"는 경고성 필드이다.


- **Private Field (`self.__변수`)**
- 이름 앞에 언더바 **2개**를 붙인다. 클래스 외부 및 자식 클래스에서의 접근을 문법적으로 원천 봉쇄한다.
- 파이썬 내부에서 이 변수의 이름을 강제로 변조(`_클래스명__변수명`)하여 숨겨버리기 때문에 외부에서 직접적인 호출이 불가능해진다.



---

### 정리

- 단일 상속의 부모 생성자는 `super().__init__()`으로 호출하는 것이 일반적이다.
- 다중 상속의 부모 생성자는 `클래스명.__init__(self)` 형태로 명시적 호출을 권장한다.
- 다중 상속 시 이름이 충돌하면 클래스 선언부 좌측(맨 앞)에 적힌 부모가 우선권(MRO)을 쥔다.
- 언더바 1개(`_`)는 개발자 간의 약속인 Protected 변수, 언더바 2개(`__`)는 문법상 원천 차단되는 Private 변수를 뜻한다.
