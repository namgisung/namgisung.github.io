## **패키지(Package)**

패키지는 여러 개의 파이썬 모듈(`.py`)을 디렉토리(폴더) 구조로 묶어서 관리하는 서브 그룹이다.

모듈의 수가 너무 많아지거나 프로그램의 규모가 커질 때, 관련 있는 기능끼리 폴더별로 분류하여 체계적으로 관리하기 위해 사용한다.

---

### **(1) 기본 구조**

점(`.`) 표기법을 사용하여 계층 구조로 모듈을 호출한다.

```python
import 패키지명.모듈명

```

* 패키지명 : 모듈들을 담고 있는 폴더 이름
* 모듈명 : 폴더 내부에 위치한 파이썬 파일명

---

### **(2) 물리적 디렉토리 구조 (예시)**

도형과 관련된 프로그램을 만든다고 가정할 때, 아래와 같이 폴더(패키지) 안에 파일(모듈)들을 분리하여 구성한다.

```text
shapes/                # 패키지 폴더 (하위 기능을 묶는 최상위 이름)
├── __init__.py        # 이 폴더가 패키지임을 파이썬에 알리는 특수 파일
├── line.py            # 선과 관련된 기능을 담은 모듈
├── triangle.py        # 삼각형과 관련된 기능을 담은 모듈
└── rectangle.py       # 사각형과 관련된 기능을 담은 모듈

```

개별 파일들이 독립적인 기능을 수행하며, 이 전체를 `shapes`라는 패키지가 감싸고 있다.

---

### **(3) 모듈 vs 패키지 비교**

개념적 스케일과 물리적 형태의 확실한 차이점이다.

| 구분 | 모듈 (Module) | 패키지 (Package) |
| --- | --- | --- |
| **물리적 형태** | 단일 파이썬 파일 (`*.py`) | 모듈들을 담고 있는 디렉토리 (폴더) |
| **포함 관계** | 변수, 함수, 클래스를 포함함 | 여러 개의 모듈 및 하위 패키지를 포함함 |
| **호출 예시** | `import calc` | `import shapes.triangle` |

---

### **(4) __init__.py 파일의 역할**

패키지 폴더 안에 존재하는 `__init__.py` 파일은 해당 폴더가 일반 폴더가 아니라 **파이썬 패키지의 일부임을 인식시키는 매개체**이다.

* 파이썬 3.3 버전부터는 이 파일이 없어도 패키지로 인식되지만, 하위 버전 호환성과 패키지 초기화 설정(예: `__all__` 변수 정의)을 명시하기 위해 여전히 생성해 두는 것이 관례이다.
* 보통은 내용물 없이 비어 있는 공문서 상태로 폴더 내에 위치시킨다.

---

### **(5) 패키지의 연결 형태**

프로그램 내부에서 메인 파일이 패키지 내 모듈을 타고 들어가는 논리적 흐름이다.

```text
[ main.py ]
    │
    └── import shapes.triangle ──> [ shapes (폴더) ]
                                       │
                                       └── [ triangle.py (파일) ]
                                               └── triangle_area() 호출

```

---

## 정리

- 패키지는 여러 모듈을 계층적으로 관리하기 위한 파이썬의 '폴더(디렉토리)' 구조이다.
- 대규모 프로젝트에서 파일 이름이 겹치는 것을 방지하고 유지보수성을 크게 높여준다.
- `패키지명.모듈명` 형태로 접근하여 내부 모듈의 기능을 가져온다.
- 폴더 내부에 `__init__.py` 파일이 존재하면 파이썬이 이를 정식 패키지로 보다 안전하게 인식한다.
