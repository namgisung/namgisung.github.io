## 배열의 출력

C 언어에서는 **배열 전체를 한 번에 출력하는 방법이 없다.**

따라서 **반드시 반복문을 사용해서 요소 하나하나 출력해야 한다.**

👉 이 차이를 이해하면 자바 vs C 배열 개념이 확실히 잡힌다.

---

### ✔️ C 배열 출력의 핵심 특징

* ✔️ **반드시 반복문 필요**
* ✔️ `Arrays.toString()` 같은 함수 없음
* ✔️ 배열 이름만 출력하면 주소값이 나옴
* ✔️ 배열 길이를 자동으로 알 수 없음

---

## (1) for문 활용 (기본적인 출력 방법)

배열을 초기화할 때 for문을 사용하듯이,

배열에 저장된 값을 확인할 때도 **for문으로 하나씩 출력해야 한다.**

```c
#include <stdio.h>

int main() {
    int iArr[5] = {100, 95, 80, 70, 60};

    for(int i = 0; i < 5; i++) {
        printf("%d\n", iArr[i]);
    }

    return 0;
}
```

각 요소가 **한 줄씩 출력**된다.

---

### 줄 바꿈 없이 출력하기

`printf`는 자동 줄 바꿈이 없으므로,

여러 값을 한 줄에 출력하려면 다음과 같이 작성한다.

```c
#include <stdio.h>

int main() {
    int iArr[5] = {100, 95, 80, 70, 60};

    for(int i = 0; i < 5; i++) {
        printf("%d ", iArr[i]);
    }
    printf("\n"); // 다음 출력이 바로 이어지지 않도록 줄 바꿈

    return 0;
}
```

출력 결과:

```
100 95 80 70 60
```

## (2) 배열을 한 번에 출력할 수 없는 이유

자바에서는 다음과 같은 코드가 가능하다.

```java
System.out.println(Arrays.toString(iArr));
```

하지만 **C 언어에는 이런 기능이 없다.**

```c
printf("%d", iArr); // ❌ 배열 출력 아님
```

👉 위 코드는 **배열의 첫 번째 요소가 아니라 주소값을 출력**한다.

### 이유

* C에서 배열 이름은 **배열 전체가 아니라 시작 주소**
* `printf`는 배열 구조를 모름
* 그래서 **개발자가 직접 반복문으로 처리해야 함**

📌 이 점이 **C 배열 이해의 핵심 포인트**

---

## (3) char 배열 출력 (문자 배열)

char 배열은 **문자열로 사용될 경우에만** 예외적으로 한 번에 출력 가능하다.

```c
#include <stdio.h>

int main() {
    char chArr[] = {'a', 'b', 'c', 'd', '\0'};

    printf("%s\n", chArr); // abcd 출력됨

    return 0;
}
```

### 왜 가능한가?

* `printf("%s")`는 **널 문자 `\0`까지를 문자열로 인식**
* 자바의 `char[]` 자동 출력과는 **완전히 다른 개념**

❗ `\0`이 없으면 쓰레기 값 출력됨 (주의)

---

## 자바 vs C 배열 출력 차이

| 구분       | Java                   | C       |
| -------- | ---------------------- | ------- |
| 배열 전체 출력 | 가능 (`Arrays.toString`) | ❌ 불가능   |
| 배열 길이    | `length` 자동 제공         | ❌ 직접 관리 |
| 배열 이름    | 객체 참조                  | 시작 주소   |
| 출력 방식    | 메서드 기반                 | 반복문 필수  |
