## 배열

배열이란 **같은 타입의 여러 변수를 하나의 묶음으로 다루는 것**이다.

예를 들어 학생 5명의 점수를 저장하기 위해서는
다음과 같이 변수 5개를 각각 선언해야 한다.

```c
int score1, score2, score3, score4, score5;
```

하지만 변수 대신 **배열**을 사용하면 다음과 같이 간단히 처리할 수 있다.
다뤄야 할 데이터 수가 아무리 많아도 **배열의 길이만 변경**하면 된다.

```c
int score[5];
```

---

### 배열의 구조 

* `score[0]`부터 `score[4]`까지 총 5개의 저장 공간이 생성된다.
* 각 저장 공간은 **모두 같은 타입(int)** 이다.
* 배열의 각 요소는 **연속된 메모리 공간**에 배치된다.

C언어에서 배열 이름 `score`는
👉 **배열 전체를 의미하는 변수가 아니라, 첫 번째 요소의 주소**를 의미한다.

즉,

```c
score == &score[0]
```

와 같은 의미를 가진다.

---

### 자바 배열과의 차이점

| 구분     | 자바       | C언어     |
| ------ | -------- | ------- |
| 배열 성격  | 객체       | 메모리 구조  |
| 배열 이름  | 참조 변수    | 시작 주소   |
| 메모리 배치 | 내부적으로 관리 | 연속된 메모리 |

자바에서는 배열이 객체이기 때문에 **참조 변수**로 다루지만,
C언어에서는 배열이 **연속된 메모리 공간 자체**이며
배열 이름은 그 **시작 주소를 가리킨다**.

---

### 배열의 특징 요약

* 동일한 데이터 타입
* 연속적인 메모리 공간
* 인덱스를 이용한 빠른 접근
* 메모리 구조를 직접 다룸

이러한 이유로 C언어 배열은
**속도는 빠르지만, 사용 시 주의가 필요한 구조**이다.
