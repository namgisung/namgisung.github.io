## 포인터 선언

포인터를 사용하려면 먼저 **포인터 변수를 선언**해야 한다.

C언어에서 포인터 선언은

**“어떤 타입의 값을 가리킬 것인가”**를 명확히 해줘야 한다.

---

### 기본 형태

```c
자료형 *포인터이름;
```

예시)

```c
int *p;
```

* `int` : 가리킬 대상의 자료형
* `*`   : 이 변수가 **포인터임을 나타냄**
* `p`   : 포인터 변수 이름

`p`는 **int형 변수의 주소만 저장할 수 있는 포인터**이다.

---

### 포인터 선언 + 초기화

포인터는 **선언만 하고 사용하면 위험**하므로

대부분 선언과 동시에 초기화한다.

```c
int a = 10;
int *p = &a;
```

* `&a` : 변수 `a`의 주소
* 포인터 `p`는 `a`를 가리키게 된다.

---

### 선언만 하고 초기화하지 않으면?

```c
int *p;
```

이 상태의 `p`는

❌ **어디를 가리키는지 알 수 없는 주소(쓰레기 값)**를 가진다.

이 상태에서 `*p`를 사용하면 **프로그램이 터질 수 있음**

(세그멘테이션 오류)

✔️ 그래서 포인터는 **선언 후 반드시 초기화**하는 습관이 중요하다.

---

### 포인터 타입이 중요한 이유

```c
int a = 10;
double b = 3.14;

int *p1 = &a;     // 가능
double *p2 = &b; // 가능

int *p3 = &b;    // ❌ 타입 불일치
```

포인터는 **가리키는 대상의 타입을 기준으로 동작**한다.

* `int*` → int 크기만큼 접근
* `double*` → double 크기만큼 접근

타입이 틀리면 **메모리 해석이 완전히 틀어짐**

---

### 여러 개의 포인터 선언 시 주의 

```c
int* p1, p2;
```

이 코드는 이렇게 해석된다.

* `p1` → int 포인터
* `p2` → 일반 int 변수

그래서 **가독성 좋은 방식**은 아래처럼 쓰는 것

```c
int *p1;
int *p2;
```
