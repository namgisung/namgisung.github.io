## 주소 연산자(&)

주소 연산자 `&`는 **변수의 메모리 주소를 얻기 위해 사용**된다.

즉,

> **“이 변수 값이 아니라, 이 변수가 저장된 위치를 달라”**
> 라고 컴퓨터에게 요청하는 연산자이다.

---

### 기본 사용법

```c
&변수이름
```

예시)

```c
int a = 10;
int *p = &a;
```

* `a`  : 값 → 10
* `&a` : 주소 → a가 저장된 메모리 위치
* `p`  : 그 주소를 저장하는 포인터

`p`는 `a`를 가리키는 포인터가 된다.

---

### 값과 주소의 차이

```c
int a = 10;

printf("%d\n", a);   // 값 출력
printf("%p\n", &a);  // 주소 출력
```

* `a`  → 값
* `&a` → 주소

주소는 정수처럼 보이지만 **메모리 위치를 의미하는 특별한 값**이다.

그래서 출력할 때 `%p` 형식 지정자를 사용한다.

---

### 포인터와 주소 연산자 관계

```c
int a = 10;
int *p;

p = &a;
```

정리하면

* `&a` → 변수 `a`의 주소
* `p`  → 주소를 저장하는 변수
* `*p` → 그 주소에 저장된 실제 값


`&` : 주소를 **가져온다**

`*` : 주소를 **따라가서 값에 접근한다**

---

### 함수 호출에서의 주소 연산자

C언어 함수는 기본적으로 **값 복사(Call by Value)** 이다.

```c
void change(int x) {
    x = 20;
}

int main() {
    int a = 10;
    change(a);
}
```

`a`는 바뀌지 않는다.

하지만 주소를 넘기면?

```c
void change(int *x) {
    *x = 20;
}

int main() {
    int a = 10;
    change(&a);
}
```

* `&a` → a의 주소 전달
* `*x` → 원래 변수 `a`에 직접 접근

✔️ 이게 **C언어에서 “값을 바꾸는 함수”의 핵심 패턴**

---

### 주소 연산자 사용 시 주의점

❌ 상수에는 주소가 없다

```c
int *p = &10; // 오류
```

❌ 연산 결과에는 주소가 없다

```c
int a = 3, b = 4;
int *p = &(a + b); // 오류
```

✔️ 주소를 가질 수 있는 것은 **변수만 가능**
