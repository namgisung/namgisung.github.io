## **1. ORDER BY절**

`ORDER BY`절은 **SELECT문으로 조회한 결과를 특정 칼럼을 기준으로 정렬**할 때 사용하는 절이다.

`ORDER BY`절은 조회 결과를 정렬한 후 출력하므로, 정렬이 필요하지 않은 경우에는 생략하여 조회 성능을 높일 수 있다.

또한 `ORDER BY`절은 **SELECT문의 가장 마지막에 실행되는 절**이며, `GROUP BY`절 이후에 수행되므로 집계 함수를 사용할 수 있다.

정렬 대상 칼럼에 `NULL` 값이 포함되어 있는 경우 DBMS마다 기본 정렬 방식이 다르다. Oracle에서는 `NULLS FIRST` 또는 `NULLS LAST` 옵션을 사용하여 NULL의 정렬 위치를 지정할 수 있다.

---

### **기본 문법**

```sql
ORDER BY 칼럼 [ASC | DESC]
```

| 옵션   | 설명        |
| ---- | --------- |
| ASC  | 오름차순(기본값) |
| DESC | 내림차순      |

> `ASC`와 `DESC`를 생략하면 기본값인 `ASC`가 적용된다.

---

### **예제 ① : 오름차순 정렬**

```sql
SELECT 번호, 이름, 주소
FROM 학생
ORDER BY 번호;
```

#### **해설**

1. `학생` 테이블에서
2. 번호, 이름, 주소를 조회한다.
3. 번호를 기준으로 오름차순 정렬하여 출력한다.

---

### **예제 ② : 내림차순 정렬**

```sql
SELECT 이름,
       수학,
       영어,
       수학 + 영어 AS 총점
FROM 성적
ORDER BY 총점 DESC;
```

#### **해설**

1. `성적` 테이블에서
2. 이름, 수학, 영어, 총점을 조회한다.
3. 총점을 기준으로 내림차순 정렬하여 출력한다.

---

## **2. ORDER BY 옵션**

| 옵션          | 설명                             |
| ----------- | ------------------------------ |
| ASC         | 오름차순으로 정렬한다.                   |
| DESC        | 내림차순으로 정렬한다.                   |
| NULLS FIRST | NULL 값을 가장 먼저 정렬한다. (Oracle)   |
| NULLS LAST  | NULL 값을 가장 마지막에 정렬한다. (Oracle) |

---

## **3. ORDER BY 특징**

| 특징             | 설명                          |
| -------------- | --------------------------- |
| 실행 순서          | SELECT문의 마지막에 실행된다.         |
| GROUP BY 이후 수행 | 집계 함수 결과를 기준으로 정렬할 수 있다.    |
| 기본 정렬          | ASC(오름차순)                   |
| 성능             | 정렬이 필요하지 않다면 생략하는 것이 효율적이다. |

---

### **추가 정리**

| 절           | 역할                   |
| ----------- | -------------------- |
| ORDER BY    | 조회 결과를 정렬한다.         |
| ASC         | 오름차순 정렬              |
| DESC        | 내림차순 정렬              |
| NULLS FIRST | NULL을 먼저 정렬(Oracle)  |
| NULLS LAST  | NULL을 나중에 정렬(Oracle) |

> SQLD에서는 **ORDER BY의 실행 순서**, **ASC와 DESC의 기본값**, **집계 함수를 이용한 정렬**, **Oracle의 NULLS FIRST / NULLS LAST**가 자주 출제된다.
