## **SELECT문**
---
### **1. SELECT문**

SELECT는 데이터베이스에서 **데이터를 조회(Retrieve)** 하는 명령어이다.

조회할 칼럼과 테이블을 지정하며, `WHERE`절을 이용하여 특정 조건에 맞는 데이터만 조회할 수 있다.

---

#### **기본 문법**

```sql
SELECT 칼럼1 [, 칼럼2, ...]
FROM 테이블명;
```

* 지정한 테이블에서 원하는 칼럼을 조회한다.
* 여러 개의 칼럼을 동시에 조회할 수 있다.

---

#### **모든 칼럼 조회**

```sql
SELECT *
FROM 회원;
```

`*`(Asterisk)는 **모든 칼럼**을 의미한다.

---

#### **특정 칼럼 조회**

```sql
SELECT 회원번호, 회원명
FROM 회원;
```

회원 테이블에서 **회원번호와 회원명**만 조회한다.

---

#### **테이블 별명(Alias)**

테이블에는 **별명(Alias)** 을 부여할 수 있으며, SQL문을 간결하게 작성할 수 있다.

```sql
SELECT A.회원번호, B.예약번호
FROM 회원 A, 예약 B
WHERE A.회원명 = B.예약자;
```

##### **해설**

1. 회원 테이블을 `A`, 예약 테이블을 `B`로 지정한다.
2. 회원명과 예약자가 같은 행을 찾는다.
3. 회원번호와 예약번호를 조회한다.

> 동일한 이름의 칼럼이 여러 테이블에 존재하는 경우 **테이블명(또는 Alias).칼럼명** 형태로 작성해야 한다.

예)

```sql
MEMBER.ID
CLIENT.ID
```

---

#### **칼럼 별명(Alias)**

조회 결과의 칼럼 이름을 변경할 수 있다.

```sql
SELECT 회원번호 AS ID, 회원명
FROM 회원;
```

`AS`는 생략 가능하다.

```sql
SELECT 회원번호 ID, 회원명
FROM 회원;
```

##### **해설**

1. 회원번호에 `ID`라는 별명을 부여한다.
2. 회원번호(ID)와 회원명을 조회한다.

---

### **2. SELECT문의 실행 순서**

SELECT문은 작성한 순서가 아니라 **실행 순서**에 따라 처리된다.

| 절        | 실행 순서 | 설명           |
| -------- | ----- | ------------ |
| FROM     | 1     | 조회할 테이블을 선택  |
| WHERE    | 2     | 조건에 맞는 행을 선택 |
| GROUP BY | 3     | 그룹 생성        |
| HAVING   | 4     | 그룹 조건 필터링    |
| SELECT   | 5     | 조회할 칼럼 선택    |
| ORDER BY | 6     | 결과 정렬        |

#### **암기법**

**F W G H S O**

> **"물에 빠진 사람을 건졌더니 욕하면서 왜 구했소."**

---

### **3. 산술 연산자**

산술 연산자는 칼럼에 대해 계산을 수행할 때 사용한다.

* 같은 행(Row)의 값끼리 계산한다.
* Null과 연산하면 결과는 Null이다.
* 0으로 나누면 오류가 발생한다.

---

#### **산술 연산자의 종류**

| 연산자 | 설명       | 우선순위 |
| --- | -------- | ---- |
| ()  | 괄호       | 1    |
| *   | 곱셈       | 2    |
| /   | 나눗셈      | 2    |
| %   | 나머지(Mod) | 2    |
| +   | 덧셈       | 3    |
| -   | 뺄셈       | 3    |

---

#### **예제**

```sql
SELECT
    수학,
    영어,
    수학 + 영어 AS 총점,
    (수학 + 영어) / 2 AS 평균
FROM 성적;
```

##### **해설**

1. 수학과 영어 점수를 조회한다.
2. 두 과목의 합을 계산하여 `총점`으로 출력한다.
3. 평균을 계산하여 `평균`으로 출력한다.

---

### **4. DUAL 테이블**

DUAL은 Oracle에서 제공하는 **가상의 테이블**이다.

실제 테이블이 없어도 함수나 계산식을 실행할 때 사용한다.

#### **예제**

```sql
SELECT 10 + 20
FROM DUAL;
```

```sql
SELECT SYSDATE
FROM DUAL;
```

---

### **5. 합성 연산자(Concatenation Operator)**

`||`는 문자열을 연결하는 **합성 연산자**이다.

문자열 또는 문자열 타입의 칼럼을 연결할 때 사용한다.

---

#### **예제**

```sql
SELECT FNAME || ' ' || LNAME AS FULL_NAME
FROM MEMBER;
```

##### **해설**

1. FNAME과 LNAME을 연결한다.
2. 가운데 공백을 추가한다.
3. FULL_NAME이라는 별명으로 조회한다.

---

### **6. 추가 정리**

| 항목         | 내용               |   |        |
| ---------- | ---------------- | - | ------ |
| `SELECT *` | 모든 칼럼 조회         |   |        |
| `AS`       | 칼럼의 별명 지정(생략 가능) |   |        |
| Alias      | 테이블 또는 칼럼의 별명    |   |        |
| `WHERE`    | 조건에 맞는 행 조회      |   |        |
| `          |                  | ` | 문자열 연결 |
| DUAL       | Oracle의 가상 테이블   |   |        |

> SQLD에서는 **SELECT문의 실행 순서(FWGHSO)**, **테이블/칼럼 Alias**, **산술 연산자**, **DUAL 테이블**, **문자열 합성 연산자(`||`)**가 자주 출제된다.
