## **1. WHERE절**

WHERE절은 **특정 조건을 만족하는 행(Row)만을 대상으로 연산을 수행**할 때 사용하는 절이다.

SELECT뿐만 아니라 UPDATE, DELETE에서도 사용할 수 있으며, INSERT에서는 사용할 수 없다.

---

### **기본 문법**

```sql
SELECT 칼럼1 [, 칼럼2, ...]
FROM 테이블명
WHERE 조건식;
```

---

### **예제 ① : SELECT**

```sql
SELECT *
FROM EMP
WHERE DEPTNO = 30;
```

#### **해설**

1. `EMP` 테이블에서
2. `DEPTNO`가 `30`인 행만 선택하여
3. 모든 칼럼을 조회한다.

---

### **예제 ② : UPDATE**

```sql
UPDATE EMP
SET JOB = 'SALESMAN'
WHERE DEPTNO = 30;
```

#### **해설**

1. `DEPTNO`가 `30`인 행의
2. `JOB` 칼럼 값을 `'SALESMAN'`으로 변경한다.

---

### **예제 ③ : DELETE**

```sql
DELETE
FROM EMP
WHERE COMM IS NULL;
```

#### **해설**

1. `COMM` 칼럼의 값이 `NULL`인 행을 삭제한다.

---

> NULL은 비교 연산자(`=`, `!=`, `<>`)로 비교할 수 없다.
> NULL 비교는 반드시 `IS NULL`, `IS NOT NULL`을 사용해야 한다.

---

## **2. 비교 연산자**

### **2.1 단일행 비교 연산자**

| 연산자 | 설명                             |
| --- | ------------------------------ |
| =   | 두 값이 같으면 참을 반환                 |
| <   | 첫 번째 값이 두 번째 값보다 작으면 참을 반환     |
| <=  | 첫 번째 값이 두 번째 값보다 작거나 같으면 참을 반환 |
| >   | 첫 번째 값이 두 번째 값보다 크면 참을 반환      |
| >=  | 첫 번째 값이 두 번째 값보다 크거나 같으면 참을 반환 |

---

### **2.2 다중행 비교 연산자**

| 연산자    | 설명                                        |
| ------ | ----------------------------------------- |
| IN     | 리스트 중 동일한 값이 하나라도 있으면 참을 반환(OR)           |
| EXISTS | 서브쿼리의 결과가 한 건이라도 있으면 참을 반환(서브쿼리에서만 사용 가능) |
| ALL    | 리스트의 모든 값과 비교하여 조건이 모두 참이면 참을 반환(AND)     |
| ANY    | 리스트의 값 중 하나라도 조건이 참이면 참을 반환(OR)           |

---

## **3. 부정 비교 연산자**

### **3.1 단일행 부정 비교 연산자**

| 연산자         | 설명              |
| ----------- | --------------- |
| !=          | 두 값이 다르면 참을 반환  |
| ^=          | 두 값이 다르면 참을 반환  |
| <>          | 두 값이 다르면 참을 반환  |
| IS NOT NULL | NULL이 아니면 참을 반환 |

---

### **3.2 다중행 부정 비교 연산자**

| 연산자        | 설명                              |
| ---------- | ------------------------------- |
| NOT IN     | 리스트 중 동일한 값이 하나도 없으면 참을 반환(AND) |
| NOT EXISTS | 서브쿼리의 결과가 한 건도 없으면 참을 반환(AND)   |

---

## **4. SQL 연산자**

| 연산자             | 설명                        |
| --------------- | ------------------------- |
| BETWEEN A AND B | A 이상 B 이하이면 참을 반환         |
| LIKE            | 패턴 문자열을 이용하여 문자를 검색할 때 사용 |

---

## **5. 부정 SQL 연산자**

| 연산자                 | 설명                        |
| ------------------- | ------------------------- |
| NOT BETWEEN A AND B | A보다 작거나 B보다 크면 참을 반환      |
| NOT LIKE            | 지정한 패턴과 일치하는 값이 없으면 참을 반환 |

---

## **6. 논리 연산자**

| 연산자 | 설명                      |
| --- | ----------------------- |
| AND | 연결된 모든 조건이 참이면 참을 반환    |
| OR  | 연결된 조건 중 하나라도 참이면 참을 반환 |
| NOT | 조건식의 참과 거짓을 반대로 반환      |

---

### **추가 정리**

| 구분          | 사용 목적                |
| ----------- | -------------------- |
| WHERE       | 조건에 맞는 행만 조회, 수정, 삭제 |
| IS NULL     | NULL 값 조회            |
| IS NOT NULL | NULL이 아닌 값 조회        |
| IN          | 여러 값 중 하나와 일치하는지 확인  |
| BETWEEN     | 범위 검색                |
| LIKE        | 문자열 패턴 검색            |

> SQLD에서는 **비교 연산자**, **IN / EXISTS / ALL / ANY**, **BETWEEN**, **LIKE**, **NULL*비교(`IS NULL`, `IS NOT NULL`)**가 자주 출제된다.
