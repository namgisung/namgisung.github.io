## 3.1 관계형 데이터베이스 개요
---

### **1. 데이터(Data)**

데이터(Data)는 컴퓨터로 어떤 일을 처리할 때 **처리의 대상이 되는 정보**를 의미한다.

데이터는 숫자뿐만 아니라 문자열, 이미지, 소리, 영상 등 다양한 형태로 존재하며, 목적에 따라 가공(Processing)되어 활용된다.

---

### **2. 데이터베이스(Database)**

데이터베이스(Database)는 데이터를 **일정한 체계에 따라 통합하여 저장한 데이터의 집합**이다.

데이터는 디스크나 메모리에 저장되며, 응용 프로그램에 종속되지 않도록 독립적으로 관리된다.

파일(File) 형태로 데이터를 관리하는 방식보다 다음과 같은 장점을 가진다.

| 장점         | 설명                              |
| ---------- | ------------------------------- |
| 자료의 독립성    | 프로그램과 데이터를 독립적으로 관리할 수 있다.      |
| 데이터 중복 최소화 | 동일한 데이터를 중복 저장하지 않는다.           |
| 통합 관리      | 여러 사용자가 동일한 데이터를 함께 사용할 수 있다.   |
| 데이터 무결성    | 데이터의 일관성과 정확성을 유지할 수 있다.        |
| 데이터 공유     | 여러 응용 프로그램에서 동일한 데이터를 사용할 수 있다. |

---

### **3. DBMS(Database Management System)**

DBMS(Database Management System)는 데이터베이스를 생성, 관리, 조회, 수정할 수 있도록 지원하는 **데이터베이스 관리 시스템**이다.

응용 프로그램과 데이터베이스 사이에서 중개 역할을 수행하며, 데이터의 저장과 관리 기능을 제공한다.

---

### **4. 관계형 데이터베이스(Relational Database)**

관계형 데이터베이스(Relational Database)는 **데이터의 일관성과 무결성을 유지하기 위해 관계(Relation)를 기반으로 데이터를 관리하는 데이터베이스 시스템**이다.

데이터를 **행(Row)과 열(Column)으로 구성된 2차원 테이블(Table)** 형태로 저장하며, 수학의 관계대수(Relational Algebra)를 기반으로 데이터를 처리한다.

---

#### **관계형 데이터베이스의 특징**

| 특징         | 설명                               |
| ---------- | -------------------------------- |
| 테이블 구조     | 데이터를 행(Row)과 열(Column) 형태로 저장한다. |
| 관계 기반      | 여러 테이블 간의 관계를 정의하여 데이터를 관리한다.    |
| 데이터 무결성    | 데이터의 일관성과 정확성을 유지한다.             |
| 데이터 중복 최소화 | 정규화를 통해 중복을 최소화한다.               |
| SQL 사용     | SQL을 이용하여 데이터를 관리한다.             |

---

### **5. 테이블(Table)**

테이블(Table)은 관계형 데이터베이스에서 데이터를 저장하는 **2차원 구조의 저장 공간**이다.

논리적 데이터 모델에서는 **엔터티(Entity)**, 물리적 데이터 모델에서는 **테이블(Table)**, 관계형 데이터베이스에서는 **릴레이션(Relation)**이라고 한다.

---

#### **행(Row)과 열(Column)**

| 구분        | 설명            | 다른 명칭            |
| --------- | ------------- | ---------------- |
| 행(Row)    | 하나의 데이터(인스턴스) | Record, Tuple    |
| 열(Column) | 데이터의 속성       | Field, Attribute |

---

#### **테이블 예시**

| 학번      | 이름  | 학과      |
| ------- | --- | ------- |
| 2026001 | 홍길동 | 컴퓨터공학   |
| 2026002 | 김철수 | 소프트웨어학과 |

---

### **6. SQL(Structured Query Language)**

SQL은 관계형 데이터베이스에서 데이터를 정의하고 조회, 입력, 수정, 삭제하기 위한 **표준 질의 언어**이다.

SQL은 크게 **DDL, DML, DCL, TCL**로 구분된다.

---

#### **SQL의 종류**

| 구분  | 의미         | 주요 명령어                                |
| --- | ---------- | ------------------------------------- |
| DDL | 데이터 정의 언어  | CREATE, ALTER, DROP, RENAME, TRUNCATE |
| DML | 데이터 조작 언어  | SELECT, INSERT, UPDATE, DELETE, MERGE |
| DCL | 데이터 제어 언어  | GRANT, REVOKE                         |
| TCL | 트랜잭션 제어 언어 | COMMIT, ROLLBACK, SAVEPOINT           |

---

#### **DDL(Data Definition Language)**

데이터베이스의 구조(스키마)를 정의하는 언어이다.

| 명령어      | 설명                          |
| -------- | --------------------------- |
| CREATE   | 객체 생성                       |
| ALTER    | 객체 수정                       |
| DROP     | 객체 삭제                       |
| RENAME   | 객체 이름 변경                    |
| TRUNCATE | 테이블의 모든 데이터를 삭제하고 구조는 유지한다. |

---

#### **DML(Data Manipulation Language)**

테이블의 데이터를 조회하거나 입력, 수정, 삭제하는 언어이다.

| 명령어    | 설명     |
| ------ | ------ |
| SELECT | 데이터 조회 |
| INSERT | 데이터 입력 |
| UPDATE | 데이터 수정 |
| DELETE | 데이터 삭제 |
| MERGE  | 데이터 병합 |

> DML은 COMMIT 이전까지 ROLLBACK이 가능하다.

---

#### **DCL(Data Control Language)**

사용자의 접근 권한과 보안을 관리하는 언어이다.

| 명령어    | 설명    |
| ------ | ----- |
| GRANT  | 권한 부여 |
| REVOKE | 권한 회수 |

---

#### **TCL(Transaction Control Language)**

트랜잭션을 제어하는 언어이다.

| 명령어       | 설명                         |
| --------- | -------------------------- |
| COMMIT    | 변경 내용을 영구 저장하고 트랜잭션을 종료한다. |
| ROLLBACK  | 변경 내용을 취소하여 이전 상태로 되돌린다.   |
| SAVEPOINT | ROLLBACK을 수행할 저장점을 지정한다.   |

---

#### **DELETE, TRUNCATE, DROP 비교**

| 구분        | DELETE | TRUNCATE   | DROP       |
| --------- | ------ | ---------- | ---------- |
| 분류        | DML    | DDL        | DDL        |
| 데이터 삭제    | O      | O          | O          |
| 테이블 구조 삭제 | X      | X          | O          |
| ROLLBACK  | 가능     | 불가능        | 불가능        |
| 로그 생성     | O      | 거의 생성하지 않음 | 거의 생성하지 않음 |
| 저장 공간 반환  | X      | O          | O          |

---

#### **추가 정리**

| 용어       | 의미                     |
| -------- | ---------------------- |
| Database | 데이터를 체계적으로 저장한 집합      |
| DBMS     | 데이터베이스를 관리하는 시스템       |
| Relation | 관계형 데이터베이스에서의 테이블      |
| Row      | 하나의 데이터(Record, Tuple) |
| Column   | 데이터의 속성(Field)         |
| SQL      | 관계형 데이터베이스를 관리하는 표준 언어 |

> SQLD에서는 **DDL·DML의 차이**, **DELETE·TRUNCATE·DROP의 차이**, **COMMIT·ROLLBACK·SAVEPOINT**, **Entity·Table·Relation의 차이**를 자주 출제한다.
