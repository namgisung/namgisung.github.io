## **2.5 본질식별자 vs 인조식별자**

|항목|설명|
|---|---|
|본질식별자|업무에 존재하는 원래의 식별자로 원조삭별자라고도 함|
|인조식별자|업무에 존재하지 않으나 원래의 식별자가 너무 복잡하게 구성되어 있어 인위적으로 만든 식별자로 대리식별자라고도 함|

### **본질식별자 모델**
```text
[도서]
- 도서번호 (PK)
- 도서명
- 저자
      |
      | (1:1 또는 1:0..1)
      v
[재고]
- 도서번호 (PK, FK -> 도서.도서번호)
- 재고수량
      |
      | (1:N)
      v
[주문목록]
- 주문번호 (PK, FK -> 주문.주문번호)
- 도서번호 (PK, FK -> 재고.도서번호)
- 구매가격
- 구매수량
      ^
      | (N:1)
      |
[주문]
- 주문번호 (PK)
- 배송지주소
- 주문일자
```
#### **인덱스 구성**
- PK: `(주문번호, 도서번호)`

### *8인조식별자 모델**
```text
[도서]
- 도서번호 (PK)
- 도서명
- 저자
      |
      | (1:1 또는 1:0..1)
      v
[재고]
- 도서번호 (PK, FK -> 도서.도서번호)
- 재고수량
      |
      | (1:N)
      v
[주문목록]
- 주문목록번호 (PK)            <-- 인조식별자
- 주문번호 (FK -> 주문.주문번호)
- 도서번호 (FK -> 재고.도서번호)
- 구매가격
- 구매수량
      ^
      | (N:1)
      |
[주문]
- 주문번호 (PK)
- 배송지주소
- 주문일자
```
#### **인덱스 구성**
- PK: `(주문목록번호)`
- IX1: `(주문번호, 도서번호)` ← 별도 생성 권장

| 구분     | 내용                                                                         |
| ------ | -------------------------------------------------------------------------- |
| **장점** | 추가적인 연산 없이 시퀀스(Sequence)나 키 제약조건 등을 통해 주식별자를 생성할 수 있으므로 개발의 편의성을 향상할 수 있다. |
| **단점** | 데이터의 중복이 발생할 수 있으며, 별도의 인덱스 생성 등이 필요하다.                                    |
