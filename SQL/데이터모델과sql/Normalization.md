## **2.1 정규화**

모델링된 데이터베이스의 테이블과 관계는 데이터를 입력, 수정, 삭제하는 과정에서 의도치 않게 일관성이 깨질 수 있으며, 이를 데이터베이스의 이상현상(Anomaly)이라고 한다.  
이러한 문제를 방지하기 위해 데이터 중복을 최소화하면서 테이블을 더 잘 조직된 상태로 분해하는 과정이 정규화(Normalization)이다.

---

### **1. 정규화 개요**

| 항목 | 내용 |
| --- | --- |
| 정규화(Normalization) | 데이터 중복 최소화 + 이상현상 방지를 위해 테이블을 분해하는 과정 |
| 핵심 목적 | 데이터 무결성/일관성 확보 |
| 기대 효과 | 입력/수정/삭제 성능 및 데이터 품질 향상 |
| 주의점 | 조회 시 조인 증가로 성능 저하 가능 |

---

### **2. 성능 영향과 반정규화**

| 구분 | 일반적 영향 | 비고 |
| --- | --- | --- |
| 입력 성능 | 향상 | 중복 감소 효과 |
| 수정 성능 | 향상 | 갱신 대상 명확 |
| 삭제 성능 | 향상 | 삭제 이상 완화 |
| 조회 성능 | 조건부 | 조인 증가 시 저하 가능 |
| 반정규화 목적 | 조회 성능 개선 | 조인 비용 완화 |

```text
정규화 수행
    │
    ├─ 입력/수정/삭제 성능: 일반적으로 향상
    │
    └─ 조회 성능:
         ├─ 일부 조건에서는 향상 가능
         └─ 일반적으로는 조인 증가로 성능 저하
                    │
                    ▼
             반정규화 수행
                    │
                    ▼
              조회 성능 개선
```

---

### **3. 데이터베이스 이상현상(Anomaly)**

| 유형 | 설명 | 대표 상황 |
| --- | --- | --- |
| 삽입 이상 (Insertion Anomaly) | 데이터 삽입 시 의도하지 않은 정보까지 함께 입력해야 하는 현상 | 하나의 사실을 넣기 위해 불필요 컬럼까지 강제 입력 |
| 갱신 이상 (Update Anomaly) | 중복 데이터 중 일부만 갱신되어 불일치가 발생하는 현상 | 동일 고객 정보가 행마다 다르게 남음 |
| 삭제 이상 (Deletion Anomaly) | 특정 데이터 삭제 시 필요한 정보까지 함께 사라지는 현상 | 이력 삭제와 함께 기준 정보 소실 |

---

### **4. 정규화 절차 요약**

| 단계 | 정규형 | 제거 대상 | 핵심 기준 | 주요 조치 |
| --- | --- | --- | --- | --- |
| 1단계 | 제1정규형 (1NF) | 반복 속성/다중값 속성 | 모든 속성은 원자값(Atomic Value) | 중복값 제거, 기본키 설정 |
| 2단계 | 제2정규형 (2NF) | 부분 함수종속성 | 복합키의 일부에만 종속 금지 | 부분 종속 속성 분리 |
| 3단계 | 제3정규형 (3NF) | 이행 함수종속성 | 일반 속성 간 종속 금지 | 이행 종속 속성 분리 |

```text
[비정규형 (UNF)]
        │  제1정규화 (1NF)
        ▼
[제1정규형 (1NF)]
        │  제2정규화 (2NF)
        ▼
[제2정규형 (2NF)]
        │  제3정규화 (3NF)
        ▼
[제3정규형 (3NF)]
```

---

### **5. 제1정규형(1NF)**

| 항목 | 내용 |
| --- | --- |
| 정의 | 모든 속성이 하나의 속성값(원자값)만 갖고, 유사 속성 반복이 없는 상태 |
| 위반 사례 | 한 컬럼에 `A,B,C`처럼 다중값 저장 / Phone1, Phone2, Phone3 반복 |
| 정규화 방식 | 반복/다중값 속성을 별도 테이블로 분해 |
| 판단 기준 | 주식별자 : 일반속성 = 1:1이어야 함 (1:M이면 1NF 대상) |

---

### **6. 제2정규형(2NF)**

| 항목 | 내용 |
| --- | --- |
| 전제 | 주식별자가 복합식별자(2개 이상 속성)인 경우 |
| 위반 원인 | 일반 속성이 PK 전체가 아닌 일부에만 종속(부분 함수종속) |
| 정규화 방식 | PK 일부에 종속된 속성을 별도 테이블로 분리 |
| 결과 | 모든 일반 속성이 PK 전체에 완전 함수종속 |

#### **함수종속성 용어 정리**

| 용어 | 의미 |
| --- | --- |
| 함수종속성 (FD) | 속성 간 논리적 종속 관계 |
| 완전 함수종속 | 일반 속성이 주식별자 전체에 종속 |
| 부분 함수종속 | 일반 속성이 주식별자 일부에만 종속 |

---

### **7. 제3정규형(3NF)**

| 항목 | 내용 |
| --- | --- |
| 위반 원인 | 일반 속성 간 함수종속성(이행 함수종속성) 존재 |
| 예시 | PK: A, 일반속성: B, C 에서 `B → C` 성립 시 C는 A에 이행 종속 |
| 정규화 방식 | 일반 속성 간 종속을 분리하여 별도 테이블 구성 |
| 결과 | PK 이외 속성 간 종속 제거 |

---

### **8. 3차 정규화 이후 엔터티 관계**

| 항목 | 내용 |
| --- | --- |
| 일반적 결과 | 3차 정규화로 분리된 엔터티들은 보통 비식별자 관계가 됨 |
| 이유 | 부모 PK가 자식의 일반속성(FK)으로만 참조되는 구조가 많아짐 |
