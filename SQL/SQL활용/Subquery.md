## **4.1 서브쿼리**

SQL문 안에는 DBMS가 제공하는 다양한 함수뿐만 아니라 **또 다른 SQL문을 포함**할 수도 있다.

이처럼 **SQL문 내부에 포함되는 또 다른 독립적인 SQL문**을 **서브쿼리(Subquery)** 라고 한다.

---

### **1. 서브쿼리의 종류**

#### **서브쿼리가 위치하는 곳에 따른 분류**

| 종류                        | 위치                          |
| ------------------------- | --------------------------- |
| 스칼라 서브쿼리(Scalar Subquery) | SELECT절의 컬럼 위치 (반환 컬럼 1개)   |
| 인라인 뷰(Inline View)        | FROM절의 테이블 위치 (반환 컬럼 2개 이상) |
| 중첩 서브쿼리(Nested Subquery)  | WHERE절, HAVING절 등의 조건식      |

---

#### **메인쿼리와의 관계에 따른 분류**

| 종류                     | 설명                     |
| ---------------------- | ---------------------- |
| 연관(Correlated) 서브쿼리    | 메인쿼리의 컬럼을 서브쿼리에서 참조한다. |
| 비연관(Uncorrelated) 서브쿼리 | 메인쿼리의 컬럼을 참조하지 않는다.    |

---

### **2. 스칼라 서브쿼리(Scalar Subquery)**

스칼라 서브쿼리는 **SELECT절처럼 컬럼이 들어가는 위치**에 사용하는 서브쿼리이다.

컬럼 위치에 들어가기 때문에 **반드시 하나의 컬럼만 반환**해야 한다.

#### **예제**

```sql
SELECT
    E.EMPNO,
    E.ENAME,
    (
        SELECT D.DNAME
        FROM DEPT D
        WHERE D.DEPTNO = E.DEPTNO
    ) AS DNAME
FROM EMP E;
```

사원(EMP) 정보를 조회하면서 부서(DEPT) 테이블에서 해당 부서명을 함께 조회한다.

#### **해설**

1. 서브쿼리에서 `DEPTNO`가 같은 `DNAME`을 조회한다.
2. 조회된 부서명을 `DNAME`이라는 별칭으로 SELECT절에 출력한다.
3. 최종적으로 사원번호, 사원명, 부서명을 함께 조회한다.

---

### **3. 인라인 뷰(Inline View)**

인라인 뷰는 **FROM절의 테이블 위치**에 사용하는 서브쿼리이다.

실행 시 일시적으로 생성되는 **가상의 테이블(View)** 처럼 동작한다.

인라인 뷰를 사용하면

* 복잡한 SQL을 단계적으로 작성할 수 있고,
* 필요한 컬럼만 먼저 추출하여 비교 대상이 줄어들므로 성능 향상에 도움이 될 수 있다.

#### **예제**

```sql
SELECT
    E.EMPNO,
    E.ENAME,
    D.DEPTNO,
    D.DNAME,
    D.LOC
FROM EMP E,
(
    SELECT DEPTNO, DNAME, LOC
    FROM DEPT
) D
WHERE E.DEPTNO = D.DEPTNO;
```

부서 테이블에서 필요한 컬럼만 먼저 추출한 후 사원 테이블과 조인한다.

#### **해설**

1. 서브쿼리에서 `DEPTNO`, `DNAME`, `LOC`만 조회한다.
2. 조회 결과에 `D`라는 별칭을 부여한다.
3. `EMP`와 `D`의 `DEPTNO`가 같은 행만 조회한다.
4. 사원번호, 사원명, 부서번호, 부서명, 지역을 출력한다.

---

### **4. 중첩 서브쿼리(Nested Subquery)**

중첩 서브쿼리는 **WHERE절이나 HAVING절 등의 조건식**에 포함되는 서브쿼리이다.

스칼라 서브쿼리와 달리 반환 형태가 다양하며, 단일행·다중행·다중컬럼을 모두 반환할 수 있다.

---

#### **반환값의 종류**

| 반환 유형              | 설명                                                                     |
| ------------------ | ---------------------------------------------------------------------- |
| 단일행(Single Row)    | 결과가 1행 이하인 경우. `=`, `>`, `<`, `>=`, `<=`, `<>` 등의 단일행 비교 연산자와 함께 사용한다. |
| 다중행(Multi Row)     | 결과가 여러 행인 경우. `IN`, `ANY`, `ALL`, `EXISTS` 등의 다중행 비교 연산자와 함께 사용한다.     |
| 다중컬럼(Multi Column) | 여러 개의 컬럼을 반환하는 경우로, 테이블 형태의 결과를 반환한다.                                  |

---

#### **예제 ① 단일행 서브쿼리**

```sql
SELECT EMPNO, ENAME, SAL
FROM EMP
WHERE SAL >= (
    SELECT AVG(E.SAL)
    FROM EMP E
);
```

급여가 평균 이상인 사원만 조회한다.

**해설**

1. 서브쿼리에서 평균 급여를 구한다.
2. 메인쿼리에서 평균 급여 이상인 사원만 조회한다.

---

#### **예제 ② 다중행 서브쿼리**

```sql
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO IN (
    SELECT DISTINCT DEPTNO
    FROM EMP
    WHERE SAL >= 1000
);
```

급여가 1000 이상인 사원이 속한 부서의 사원 정보를 조회한다.

**해설**

1. 서브쿼리에서 급여가 1000 이상인 사원의 부서번호를 중복 없이 조회한다.
2. 메인쿼리에서 해당 부서번호에 속하는 사원을 조회한다.

---

#### **예제 ③ 다중컬럼 서브쿼리**

```sql
SELECT EMPNO, ENAME, JOB, SAL
FROM EMP
WHERE (JOB, SAL) IN (
    SELECT JOB, MAX(SAL)
    FROM EMP
    GROUP BY JOB
);
```

직무(Job)별 최고 급여를 받는 사원을 조회한다.

**해설**

1. 서브쿼리에서 직무별 최고 급여를 구한다.
2. 메인쿼리에서 직무와 급여가 모두 일치하는 사원을 조회한다.

---

### **5. 테이블(Table)과 뷰(View)의 차이**

| 구분       | 테이블(Table)      | 뷰(View)          |
| -------- | --------------- | ---------------- |
| 개념       | 실제 데이터를 저장하는 객체 | 테이블을 참조하는 가상의 객체 |
| 데이터 저장   | 실제 데이터를 저장      | 실제 데이터를 저장하지 않음  |
| 생성 방식    | 정적(Static) 생성   | 동적(Dynamic) 생성   |
| 인덱스      | 자체 인덱스 생성 가능    | 자체 인덱스 생성 불가     |
| 삽입/수정/삭제 | 제한 없음           | 제약이 있음           |

---

### **6. 뷰(View)**

뷰(View)는 **하나 이상의 테이블을 기반으로 생성되는 가상의 테이블**이다.

실제 데이터를 저장하지 않고, 원본 테이블의 데이터를 조회하여 보여준다.

---

#### **뷰의 특징**

| 특징  | 설명                                       |
| --- | ---------------------------------------- |
| 편리성 | 복잡한 SQL을 뷰로 만들어 재사용할 수 있다.               |
| 보안성 | 필요한 컬럼만 노출하여 데이터를 보호할 수 있다.              |
| 독립성 | 테이블 구조가 변경되어도 뷰가 유지되면 응용 프로그램 수정이 최소화된다. |

---

#### **뷰 생성**

```sql 
CREATE OR REPLACE VIEW V_EMP AS
SELECT
    EMPNO,
    ENAME,
    SAL,
    E.DEPTNO,
    D.DNAME,
    D.LOC
FROM EMP E
LEFT OUTER JOIN DEPT D
ON E.DEPTNO = D.DEPTNO;
```

* `V_EMP`라는 뷰를 생성한다.
* `CREATE OR REPLACE`는 같은 이름의 뷰가 없으면 생성하고, 있으면 기존 뷰를 대체한다.
* `LEFT OUTER JOIN`을 사용하여 EMP의 모든 데이터를 조회한다.

---

#### **뷰 조회**

```sql
SELECT *
FROM V_EMP;
```

`V_EMP`의 모든 컬럼을 조회한다.

---

#### **뷰 삭제**

```sql
DROP VIEW V_EMP;
```

`V_EMP` 뷰를 삭제한다.

---

### **정리**

| 구분       | 특징                      |
| -------- | ----------------------- |
| 스칼라 서브쿼리 | SELECT절에서 사용, 1개의 컬럼 반환 |
| 인라인 뷰    | FROM절에서 사용, 가상의 테이블 생성  |
| 중첩 서브쿼리  | WHERE/HAVING절에서 사용      |
| 연관 서브쿼리  | 메인쿼리의 컬럼을 참조            |
| 비연관 서브쿼리 | 메인쿼리의 컬럼을 참조하지 않음       |
| View     | 실제 데이터를 저장하지 않는 가상의 테이블 |
