## **1.5 식별자(Identifier)**

-----

### **1. 식별자의 개념**

| 항목 | 내용 |
| --- | --- |
| 식별자(Identifier) | 엔터티 인스턴스를 유일하게 구별할 수 있는 속성 |
| 의미 | 해당 엔터티 인스턴스의 대표 속성 |

-----

### **2. 주식별자(PK)**

주식별자(PK)는 엔터티 인스턴스를 유일하게 구별해 주는 식별자이다.

| 성질 | 설명 |
| --- | --- |
| 유일성 | 각 엔터티 인스턴스를 유일하게 구별할 수 있어야 한다. |
| 최소성 | 유일성을 보장하면서도 최소 개수의 속성이어야 한다. |
| 불변성 | 최초 생성 시 부여된 값이 변경되지 않고 유지되어야 한다. |
| 존재성 | 반드시 값을 가져야 하며 Null 값을 가질 수 없다. |

PK는 유일성과 존재성을 가져야 하므로 **Unique Key + Not Null** 제약조건을 갖는다.

-----

### **3. 식별자의 분류**

#### **1. 대표성 여부에 따른 분류**

| 구분 | 설명 |
| --- | --- |
| 주식별자(Primary Identifier) | 엔터티 인스턴스를 유일하게 구별할 수 있는 대표 식별자이며, 다른 엔터티와 참조관계를 연결할 수 있다. |
| 보조식별자(Alternate Identifier) | 엔터티를 유일하게 구별할 수 있으나 대표성은 없고 일반적으로 참조관계 연결용으로 사용되지 않는다. |
| 후보키(Candidate Key) | 주식별자(PK)와 보조식별자(AK)는 모두 후보키이며, 유일성과 최소성을 만족한다. |

#### **2. 스스로 생성 여부에 따른 분류**

| 구분 | 설명 |
| --- | --- |
| 내부식별자(Internal Identifier) | 엔터티 내부에서 스스로 만들어지는 식별자 |
| 외부식별자(Foreign Identifier) | 관계를 통해 다른 엔터티로부터 받아오는 식별자(FK) |

#### **3. 속성의 수에 따른 분류**

| 구분 | 설명 |
| --- | --- |
| 단일식별자(Single Identifier) | 식별자를 구성하는 속성이 하나인 식별자 |
| 복합식별자(Composite Identifier) | 식별자를 구성하는 속성이 둘 이상인 식별자 |

#### **4. 대체 여부에 따른 분류**

| 구분 | 설명 |
| --- | --- |
| 본질식별자(Original Identifier) | 업무에 원래 존재하는 식별자(원조식별자) |
| 인조식별자(Surrogate Identifier) | 원조식별자가 복잡할 때 인위적으로 만든 식별자(대리식별자) |

-----

### **4. 식별자 관계와 비식별자 관계**

| 구분 | 설명 | ERD 표현 | 생명주기 |
| --- | --- | --- | --- |
| 식별자 관계 | 부모 엔터티의 식별자가 자식 엔터티의 주식별자 구성에 포함되는 강한 관계 | 실선 | 부모와 자식이 같은 생명주기를 가짐 |
| 비식별자 관계 | 부모 엔터티의 식별자가 자식 엔터티의 일반 속성(FK)으로 들어가는 약한 관계 | 점선 | 부모와 자식이 서로 다른 생명주기를 가짐 |

#### **ERD 도식 예시**

##### **1) 식별자 관계 (실선)**

```text
[주문] |──────< [주문상세]
 PK: 주문ID        PK: 주문ID + 상품순번
```

해석:
- 부모(주문)의 PK가 자식(주문상세)의 PK 구성에 포함된다.
- 강한 연결이며 실선으로 표현한다.

##### **2) 비식별자 관계 (점선)**

```text
[고객] O┄┄┄┄┄< [문의]
 PK: 고객ID         PK: 문의ID, FK: 고객ID
```

해석:
- 부모(고객)의 PK가 자식(문의)의 일반 속성(FK)으로만 들어간다.
- 약한 연결이며 점선으로 표현한다.
- 부모 참조값이 없더라도 업무 규칙에 따라 자식 엔터티 생성이 가능할 수 있다.
