## **1.2 엔터티**

-----

### **1. 엔터티의 개념**

| 항목 | 내용 |
| --- | --- |
| 엔터티(Entity) | 데이터베이스의 구성요소 중 독립적으로 식별 가능한 객체(Object) |
| 속성(Attribute) | 엔터티를 상세히 표현하기 위한 하위요소 |
| 주식별자(PK) | 엔터티를 유일하게 식별하는 속성(Primary Key) |
| ERD 표현 | PK 속성과 일반 속성은 칸막이로 구분 |

-----

### **2. 엔터티의 특징**

| 번호 | 특징 |
| --- | --- |
| 1 | 업무에서 필요로 하고 관리하고자 하는 정보 |
| 2 | 식별이 가능하도록 유일한 식별자를 가져야 함 |
| 3 | 영속적으로 존재하는 인스턴스가 두 개 이상인 집합을 이루어야 함 |
| 4 | 하위요소는 반드시 속성을 가져야 함 |
| 5 | 다른 엔터티와 한 개 이상의 관계를 가져야 함 (단, 통계성/코드성 엔터티는 관계 생략 가능) |

-----

### **3. 엔터티의 분류**

#### **1. 발생시점/상속 관계에 따른 분류**

| 구분 | 설명 |
| --- | --- |
| 기본 엔터티 | 자신의 고유한 주식별자를 가지는 독립적으로 생성되는 엔터티 |
| 중심 엔터티 | 기본 엔터티로부터 주식별자를 상속받아 생성되며 업무의 중심 역할을 하는 엔터티 |
| 행위 엔터티 | 두 개 이상의 엔터티를 상속받아 생성되는 엔터티로, 내용이 자주 변경되거나 데이터 양이 계속 증가함 |

#### **2. 물리적 형태의 존재 여부에 따른 분류**

| 구분 | 설명 |
| --- | --- |
| 유형 엔터티 | 물리적 형태가 존재하는 엔터티 |
| 개념 엔터티 | 물리적 형태 없이 개념적으로 정의되는 엔터티 |
| 사건 엔터티 | 업무나 이벤트 수행 시 발생하는 행위나 이벤트를 나타내는 엔터티 |

-----

### **4. 교차 엔터티**

| 항목 | 내용 |
| --- | --- |
| 교차 엔터티 | M:N 관계를 해소하기 위해 만들어진 엔터티 |

-----

### **5. 엔터티 명명법**

| 번호 | 규칙 |
| --- | --- |
| 1 | 엔터티 생성 의미대로 이름을 부여하며 간결하고 명확하게 표현 |
| 2 | 가능하면 약어를 사용하지 않음 (단, 너무 길면 표준화된 약어 사용 가능) |
| 3 | 영문의 경우 대문자를 사용하며 단수 명사를 사용 |
| 4 | 협업의 업무 용어를 사용하여 업무상의 의미를 분명하게 함 |
| 5 | 모든 엔터티의 이름은 서로 구별될 수 있도록 유일해야 함 |
| 6 | 의미상 중복이 없도록 함 |
