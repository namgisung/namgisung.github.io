## **1.3 속성(Attribute)**

-----

### **1. 속성의 개념**

| 항목 | 내용 |
| --- | --- |
| 속성(Attribute) | 엔터티에 대한 자세하고 구체적인 정보를 나타내는 엔터티의 하위요소 |
| 설계 원칙 | 속성의 개수 제한은 없지만, 업무 프로세스에 꼭 필요한 속성으로 최소화해야 함 |
| 데이터 단위 | 속성은 의미상 더 이상 분리되지 않는 최소의 데이터 단위 |

-----

### **2. 속성값**

| 항목 | 내용 |
| --- | --- |
| 속성 | 자신이 가질 수 있는 속성값들의 집합 |
| 속성값 | 속성이 가질 수 있는 특정 값 |
| 기본 원칙 | 하나의 엔터티 인스턴스에서 각 속성은 한 개의 속성값만 가져야 함 |
| 예외 처리 | 한 개 이상의 속성값을 가질 수 있는 경우(문자열) 1차 정규화를 수행하여 한 개의 속성값을 갖도록 해야 함 |

-----

### **3. 엔터티, 인스턴스, 속성값**

| 구분 | 설명 |
| --- | --- |
| 엔터티와 인스턴스 | 한 개의 엔터티는 두 개 이상의 인스턴스를 가진다. (엔터티는 인스턴스의 집합) |
| 엔터티와 속성 | 한 개의 엔터티는 두 개 이상의 속성을 가진다. (엔터티는 속성의 집합) |

-----

### **4. 속성의 분류**

#### **1. 속성의 특성에 따른 분류**

| 구분 | 설명 |
| --- | --- |
| 기본 속성 | 엔터티가 본래부터 가지고 있어야 하는 속성 |
| 설계 속성 | 본래 속성은 아니지만 설계 시 필요하다고 판단되어 도출된 속성 |
| 파생 속성 | 다른 속성으로부터 계산되거나 특정 규칙에 따라 변형되어 만들어진 속성 |

#### **2. 속성의 구성방식에 따른 분류**

| 구분 | 설명 |
| --- | --- |
| PK(Primary Key) 속성 | 해당 엔터티의 인스턴스를 유일하게 식별할 수 있는 속성 |
| FK(Foreign Key) 속성 | 관계를 통해 다른 엔터티의 속성을 가져와 포함시킨 속성 |
| 일반 속성 | 나머지 일반 속성 |

-----

### **5. 도메인**

| 항목 | 내용 |
| --- | --- |
| 도메인 | 속성이 가질 수 있는 값의 범위(값의 데이터 타입과 크기)를 정의한 것 |
